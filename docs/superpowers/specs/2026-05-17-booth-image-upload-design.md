# 부스 이미지 업로드 — 이용자 페이지 규격 대응 설계

작성일: 2026-05-17
상태: 설계 완료, 구현 계획 단계 진입 예정
주 변경 파일: `src/features/booths/components/booth-info-form.tsx`

## 배경 / 문제

부스 운영자(Booth 역할)가 자기 부스 상세 정보를 작성할 때 이미지를 등록한다. 현재 구현(`BoothInfoForm`)의 문제:

1. **편집 모드에서만 보임.** "부스 썸네일" 업로드 섹션이 `{isEditing && (...)}` 안에 있어, 비편집(읽기 전용) 화면에는 이미지 영역 자체가 렌더되지 않는다. `booth1`처럼 프로필 작성이 완료된 계정으로 로그인하면 부스 상세 정보가 읽기 전용으로 열리므로 업로드 섹션이 보이지 않고 → "이미지 업로드 기능이 없다"고 오인된다.
2. **규격 가이드 없음.** 미리보기가 정사각형(`aspect-square`)이라, 이용자 페이지가 기대하는 비율과 어긋난다.

이용자 페이지가 기대하는 부스 이미지 규격(원 요청 출처):

| 용도 | 비율 | 기준 픽셀 |
|---|---|---|
| 상세 페이지 배너 | 3:2 | 390×260 |
| 목록 페이지 카드 썸네일 | 5:6 | 80×96 |

목표: **3:2 기준으로 이미지를 등록하되, 5:6 썸네일 크롭 영역을 미리보기로 표시**한다.

## 결정 사항 (브레인스토밍 합의)

- **데이터 모델은 변경하지 않는다.** `thumbnails: BoothImage[]` + `isMain` 다중 갤러리 구조를 그대로 유지한다. 이용자 페이지가 추후 갤러리(비대표 이미지)를 쓸 여지를 남긴다. `isBoothInfoCompleted`의 완료 조건(`thumbnails.some(isMain)`)도 현행 유지.
- **크롭 위치는 중앙 고정.** 원 요청이 "미리 표시"까지이므로 크롭 위치 드래그/조정은 범위 밖. 5:6 영역은 3:2 박스 중앙에 고정되며 미리보기 전용이다. 백엔드는 crop 좌표를 저장하지 않고, 이용자 앱이 center-crop 한다.
- **백엔드 계약 변경 없음.** 실제 업로드 엔드포인트는 앱 전체가 TBD 상태라 mock은 기존 `URL.createObjectURL` blob 방식을 유지한다. 저장은 기존대로 `updateMutation`에 `thumbnails`를 실어 보낸다.

## 설계

### 1. 5:6 크롭 영역의 기하

3:2 박스(가로:세로 = 1.5) 안에 5:6 영역(가로:세로 = 0.8333)을 세로 꽉 차게 중앙 배치하면:

- 크롭 폭 비율 = (5/6) ÷ (3/2) = **0.5556** → 3:2 박스 가로의 약 55.6%
- 크롭 높이 = 박스 세로 100%
- 좌우 각각 (1 − 0.5556) / 2 = **22.2%**가 잘려나가는 영역

즉 3:2 미리보기 위에 **가운데 55.6% 폭의 세로 띠**가 "목록 카드에 보이는 영역"이고, 좌우 22.2%씩이 목록에서 잘린다.

### 2. 업로드 UI 변경 (`BoothInfoForm` 편집 모드)

- **드롭존**: 안내 문구에 권장 규격 추가 — "권장 390×260 (3:2 비율)".
- **이미지 그리드 미리보기**: `aspect-square` → `aspect-[3/2]`로 교체. 모든 이미지가 대표 후보이므로 일괄 3:2로 표시한다.
- **대표(`isMain`) 이미지 위 5:6 크롭 오버레이**: 대표 이미지에만 표시.
  - 좌우 22.2%씩 반투명 어두운 마스크(예: `bg-black/45`)로 덮어 "잘리는 영역"을 시각화.
  - 가운데 55.6% 영역 경계에 점선 테두리.
  - 작은 라벨: "목록 카드 썸네일 영역". 별도 안내 텍스트: "핵심 피사체를 가운데에 두세요".
  - 기존의 `대표` 배지/`대표로 설정`/삭제 hover 액션과 시각적으로 겹치지 않도록 배치(오버레이는 `pointer-events-none`, 기존 버튼들이 위에 오도록 z-index 정리).

### 3. 읽기 전용 모드 이미지 표시

비편집 화면에도 이미지 그리드를 렌더한다 (현재는 아예 없음).

- 등록된 이미지가 있으면: 3:2 그리드로 표시. 대표 이미지에 `대표` 배지 + 5:6 크롭 오버레이 동일하게 표시(운영자가 편집하지 않고도 어떻게 잘리는지 확인 가능). hover 액션(대표 설정/삭제) 없음.
- 이미지가 없으면: placeholder ("등록된 이미지가 없습니다").
- 다른 읽기 전용 필드와 동일하게 "부스 소개글" 아래쯤, 일관된 위치에 배치.

### 4. 검증 / 안내

원본 이미지의 비율·크기는 **제한하지 않는다.** 원본이 어떤 비율이든 등록 표시만 3:2(배너) / 5:6(썸네일)으로 `object-cover` 크롭된다. 따라서 비율 불일치 경고는 두지 않고, 어떻게 표시되는지를 안내 문구로만 알린다.

`handleImageUpload`에 추가:

- **용량 상한**: 파일당 5MB 초과 시 해당 파일 제외 + `toast.error`로 알림.
- **타입**: 기존 `accept="image/*"` 유지. 비이미지 방어적으로 한 번 더 체크.

안내 문구:

- 드롭존: "상세 배너는 3:2, 목록 썸네일은 5:6 비율로 표시됩니다 · 원본 크기는 제한 없음 · 파일당 최대 5MB · 여러 장 선택 가능".
- 이미지 그리드 위 문단: 대표 이미지가 상세 배너로 쓰이고 가운데 점선 영역만 목록 썸네일에 보인다는 설명.

### 크롭 오버레이 컴포넌트 분리

크롭 마스크+점선+라벨 묶음은 편집 모드와 읽기 전용 모드 양쪽에서 쓰이므로 작은 프레젠테이션 컴포넌트(`ThumbnailCropOverlay` 같은)로 분리해 중복을 막는다. 입력은 없음(순수 시각 오버레이), 부모가 3:2 컨테이너 안에 절대 배치한다.
